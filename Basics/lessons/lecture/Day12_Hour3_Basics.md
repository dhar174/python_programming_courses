# Session 12, Hour 3: Capstone Polish & Demo Readiness

## 🎯 Outcomes
- Finalize capstone quality.
- Prepare a short demo walkthrough.

## 🗣️ Instructor Speaking Script

### Introduction (10-15 min)
"You've built your application—congratulations! Now it's time for the final polish. The last 10% of a project often distinguishes a good app from a great one."

"Look at your user experience. Are your prompts clear? Is there helpful text explaining what the user should enter? Have you considered edge cases, like what happens if a user types 'A' instead of '1' for a menu choice?"

"Your project also needs instructions. You should add a `README.txt` or `README.md` file explaining how to run the app and what it does. Finally, you need to prepare for your demo. Your demo must showcase your app from scratch: starting it, adding data, listing it, searching, exiting, and loading it again."

### Live Demo (5-10 min)
*Action: Share your screen with your Capstone MVP.*

"A good demo is quick and tells a story. Let me show you what a 3-minute demo looks like."

*Action: Run the app and talk through it.*
"Here is my Personal Organizer. First, I'll start the app. You can see my menu. Let's add a task for 'Finalizing Python materials'. Great, it saved. Now I'll list my tasks to prove it's there. Let's quit the app and restart it... and as you can see, 'Finalizing Python materials' is still loaded. If I make a typo here, notice how my app catches the error gently instead of crashing."

"That's exactly what you need to prepare."

## 💻 Hands-On Lab: Capstone Polish (25-35 min)

**Prompt for Students:**
Finalize your Capstone project and prepare your presentation.

**Required:**
- Add a `README.txt` or `README.md` with run instructions.
- Add a sample data file (optional, but helpful for the demo).
- Practice a full demo script: Start → Add → List → Search → Exit/Save → Restart/Load.

**Deliverable this hour:** A polished app that runs cleanly from scratch, and a rehearsed demo flow that takes under 3 minutes.

### ✅ Completion Criteria
- Program runs cleanly from scratch without crashes.
- The student's demo flow works efficiently in under 3 minutes.

### 🛑 Common Pitfalls (Instructor Eyes Only)
- **Last-minute refactoring:** Students might try to 'clean up' working code right before the demo and break it. Tell them to back up their working file first!
- **Not testing a fresh run:** Forgetting to test the app from a clean state (e.g., deleting the JSON file to see if it generates properly).

### 🚀 Optional Extensions
- Add a dedicated 'Help' screen option to the menu.

## 📝 Quick Check / Exit Ticket
**Ask the class:** "What will you show first in your demo, and why?"
